# Java 정렬 알고리즘 

## 1. 알고리즘 한눈에 보기

| 정렬 방식 | 핵심 메커니즘 | 최선 복잡도 | 최악 복잡도 | 특징 |
| :--- | :--- | :---: | :---: | :--- |
| **선택 (Selection)** | 최댓값을 찾아 맨 뒤 데이터와 교환 | O(n^2) | O(n^2) | 정렬 상태와 상관없이 항상 전체 비교 발생 |
| **버블 (Bubble)** | 인접한 두 값을 비교하며 큰 값을 뒤로 이동 | O(n^2) | O(n^2) | 매 턴마다 가장 큰 값이 맨 뒤에 고정됨 |
| **삽입 (Insertion)** | 데이터를 앞쪽의 정렬된 제자리에 삽입 | O(n) | O(n^2) | 뒤에서부터 역방향 탐색 및 시프팅이 효율적 |
| **병합 (Merge)** | 분할 정복을 통해 배열을 쪼갠 후 합치며 정렬 | O(n log n) | O(n log n) | 데이터 상태와 무관하게 항상 일정한 성능 보장 (단, 임시 배열 공간 필요) |
| **퀵 (Quick)** | 피벗 기준으로 작은 값/큰 값을 양쪽으로 분할 후 재귀 정렬 | O(n log n) | O(n^2) | 추가 메모리 없이 제자리(in-place) 정렬, 평균적으로 가장 빠름 (피벗 선택이 성능을 좌우) |

## 2. 알고리즘별 핵심 포인트

### 선택 정렬
* 정렬되지 않은 영역에서 최댓값의 인덱스를 기억했다가 맨 뒤와 스왑(Swap)합니다.

### 버블 정렬
* 인접 원소끼리 계속 자리를 바꾸면서 큰 원소를 거품처럼 뒤로 밀어냅니다.

### 삽입 정렬
* 앞에서부터 탐색하면 뒤쪽 데이터를 매번 밀어야 하므로, **뒤에서부터 앞으로 탐색**하며 공간을 한 칸씩 확보(Shifting)한 뒤 끼워 넣습니다.
* 이미 정렬된 상태(최선의 경우)에는 직전 값과 한 번만 비교하므로 O(n)이라는 최고의 효율을 냅니다.

### 병합 정렬

* **분할 정복(Divide and Conquer):** 문제를 더 이상 쪼갤 수 없을 때까지 반으로 나눈 후(`mergeSort`), 쪼개진 원소들을 정렬하며 다시 하나로 합칩니다(`merge`).
* **안정 정렬(Stable Sort):** 동일한 값을 가진 원소들의 상대적인 순서가 정렬 후에도 그대로 유지되는 안전성을 가집니다.
* **메모리 최적화:** `merge` 단계에서 매번 임시 배열(`tmp`)을 생성하면 불필요한 메모리 할당/해제가 반복되므로, 최초 호출 시 배열을 딱 한 번만 생성하여 매개변수로 공유하는 구조가 가장 효율적입니다.

### 퀵 정렬

* **분할 정복(Divide and Conquer):** 기준값인 **피벗(pivot)** 을 하나 정해, 피벗보다 작은 값은 왼쪽 / 큰 값은 오른쪽으로 모으는 **분할(`partition`)** 을 거친 뒤, 양쪽 영역을 각각 재귀적으로 정렬합니다. 분할이 끝나면 피벗은 항상 자신의 **최종 위치**에 놓입니다.
* **제자리 정렬(In-place):** 병합 정렬과 달리 별도의 임시 배열 없이 스왑(Swap)만으로 정렬하므로 추가 메모리 부담이 거의 없습니다.
* **피벗 선택이 성능을 좌우:**
    * 피벗을 항상 **맨 끝 원소**로 고정하면, 이미 정렬된 입력에서 한쪽으로만 쪼개져 **최악 O(n^2)** 가 됩니다.
    * **랜덤 피벗(`partitionRandom`)** 으로 고르면 특정 입력에 치우치지 않아, 어떤 입력에서도 평균 **O(n log n)** 성능을 안정적으로 기대할 수 있습니다.
* **불안정 정렬(Unstable Sort):** 분할 과정에서 멀리 떨어진 원소끼리 교환되므로, 동일한 값의 상대적 순서가 보장되지 않습니다. (병합 정렬과의 차이점)

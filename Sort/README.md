# Java 정렬 알고리즘 

## 1. 알고리즘 한눈에 보기

| 정렬 방식 | 핵심 메커니즘 | 최선 복잡도 | 최악 복잡도 | 특징 |
| :--- | :--- | :---: | :---: | :--- |
| **선택 (Selection)** | 최댓값을 찾아 맨 뒤 데이터와 교환 | O(n^2) | O(n^2) | 정렬 상태와 상관없이 항상 전체 비교 발생 |
| **버블 (Bubble)** | 인접한 두 값을 비교하며 큰 값을 뒤로 이동 | O(n^2) | O(n^2) | 매 턴마다 가장 큰 값이 맨 뒤에 고정됨 |
| **삽입 (Insertion)** | 데이터를 앞쪽의 정렬된 제자리에 삽입 | O(n) | O(n^2) | 뒤에서부터 역방향 탐색 및 시프팅이 효율적 |

---

## 2. 알고리즘별 핵심 포인트

### 선택 정렬
* 정렬되지 않은 영역에서 최댓값의 인덱스를 기억했다가 맨 뒤와 스왑(Swap)합니다.

### 버블 정렬
* 인접 원소끼리 계속 자리를 바꾸면서 큰 원소를 거품처럼 뒤로 밀어냅니다.

### 삽입 정렬
* 앞에서부터 탐색하면 뒤쪽 데이터를 매번 밀어야 하므로, **뒤에서부터 앞으로 탐색**하며 공간을 한 칸씩 확보(Shifting)한 뒤 끼워 넣습니다.
* 이미 정렬된 상태(최선의 경우)에는 직전 값과 한 번만 비교하므로 O(n)이라는 최고의 효율을 냅니다.
